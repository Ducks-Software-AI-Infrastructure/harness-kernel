# Code of Conduct

## Our Pledge

We want Harness Kernel to be a respectful, practical, and safe place to build
open source software.

Everyone participating in this project is expected to help keep discussions
focused, constructive, and welcoming, regardless of experience level, identity,
background, location, or affiliation.

## Expected Behavior

Examples of behavior that supports a healthy project:

- be respectful and direct;
- assume good intent while still asking for clarity;
- give technical feedback with concrete reasoning;
- keep issues and pull requests focused on the work;
- accept maintainer decisions about scope and direction;
- acknowledge mistakes and correct them when possible;
- avoid sharing private information, credentials, logs, or user data.

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment, intimidation, or personal attacks;
- insults, slurs, or discriminatory language;
- sexualized comments or imagery;
- publishing private information without permission;
- sustained disruption of issues, pull requests, discussions, or reviews;
- bad-faith arguments, trolling, or repeated derailment;
- threats of violence or harm;
- sharing secrets, credentials, private infrastructure details, or private user
  data.

## Scope

This Code of Conduct applies in all project spaces, including:

- GitHub issues, pull requests, discussions, and reviews;
- project documentation and examples;
- release, security, and maintainer communication;
- public spaces where someone is clearly representing the project.

## Reporting

If you experience or witness unacceptable behavior, report it privately by
emailing:

```text
duckssoftwarecontact@gmail.com
```

For security vulnerabilities, follow `SECURITY.md` instead of opening a public
issue.

Reports should include as much context as you can safely provide:

- where the behavior happened;
- who was involved;
- what happened;
- links, screenshots, or timestamps when relevant;
- whether there is any immediate safety or privacy concern.

Do not include secrets, credentials, private keys, or private user data in a
report unless the maintainers explicitly request a safer channel for that
information.

## Enforcement

Maintainers may take action to protect the project and its contributors,
including:

- asking for clarification or a change in behavior;
- editing, hiding, or deleting comments;
- closing issues or pull requests;
- limiting participation in a discussion;
- blocking or banning users from project spaces;
- contacting platform administrators when needed.

Enforcement decisions are made by maintainers based on the impact of the
behavior, the available context, and the safety of the project community.

## Maintainer Responsibilities

Maintainers are responsible for applying this Code of Conduct consistently and
fairly. Maintainers should avoid conflicts of interest where possible and should
not use enforcement to silence good-faith technical disagreement.

Technical disagreement is welcome. Harassment, abuse, and unsafe disclosure are
not.
